# EBS (Elastic Block Store)

EBS (Elastic Block Store) হলো আপনার EC2 সার্ভারের হার্ড ড্রাইভ বা SSD।

আপনি যখন একটি ল্যাপটপ বা পিসি কেনেন, তাতে যেমন একটি হার্ড ড্রাইভ থাকে যেখানে Windows বা Linux ইনস্টল করা থাকে এবং আপনার ফাইলগুলো জমা থাকে, EC2 এর ক্ষেত্রে সেই ড্রাইভটিই হলো EBS।

## ১. EBS কীভাবে কাজ করে?
আপনি যখন আপনার উবুন্টু সার্ভারটি লঞ্চ করেছিলেন, তখন আপনাকে একটি ৮ বা ২০ জিবির স্টোরেজ নিতে হয়েছিল। সেটিই ছিল EBS।

- Network Attached: এটি ফিজিক্যালি আপনার EC2 সার্ভারের ভেতর থাকে না। এটি একটি আলাদা নেটওয়ার্কের মাধ্যমে আপনার সার্ভারের সাথে যুক্ত থাকে (যেমন আপনার পিসিতে লাগানো পেনড্রাইভ বা এক্সটার্নাল হার্ড ড্রাইভ)।

- Persistence: আপনি যদি আপনার EC2 সার্ভারটি Stop করেন, তবে আপনার অপারেটিং সিস্টেম এবং ফাইলগুলো মুছে যায় না। কারণ EBS ড্রাইভটি তার ভেতরে ডেটাগুলো সেভ করে রাখে।

## ২. EBS-এর প্রধান বৈশিষ্ট্যসমূহ

- Availability Zones (AZ): একটি EBS ড্রাইভ সাধারণত একটি নির্দিষ্ট জোন (যেমন: us-east-1a) এর মধ্যেই থাকে। আপনার সার্ভার এবং EBS ড্রাইভ একই জোনে থাকতে হবে, নাহলে তারা একে অপরের সাথে কানেক্ট হতে পারবে না।

- Snapshots (Backup): EBS-এর সবচেয়ে শক্তিশালী ফিচার হলো স্ন্যাপশট। আপনি চাইলে আপনার হার্ড ড্রাইভের একটি ফটো বা ব্যাকআপ তুলে রাখতে পারেন। আপনার তৈরি করা AMI এর ভেতরেও এই স্ন্যাপশট থাকে।

- Elasticity: আপনি চাইলে সার্ভার বন্ধ না করেই ড্রাইভের সাইজ (যেমন ১০ জিবি থেকে ৫০ জিবি) বাড়িয়ে নিতে পারেন।

## ৩. EBS-এর প্রকারভেদ (Types)
আপনার বিলের দিকে তাকালে দেখবেন বিভিন্ন টাইপের স্টোরেজ চার্জ আসতে পারে। প্রধানত ৪ ধরণের EBS আছে:
- gp3 / gp2 (General Purpose SSD): এটি সবচেয়ে জনপ্রিয়। আপনার প্র্যাকটিসের জন্য এটিই সেরা।
- io2 / io1 (Provisioned IOPS SSD): হাই-পারফরম্যান্স ডাটাবেসের জন্য ব্যবহৃত হয়।
- st1 (Throughput Optimized HDD): বড় ডাটা প্রসেসিং এর জন্য।
- sc1 (Cold HDD): অনেক ডেটা কিন্তু খুব কম ব্যবহার হয় এমন কাজের জন্য।


# AWS AMI (Amazon Machine Image)

AWS AMI (Amazon Machine Image) হলো একটি ভার্চুয়াল অ্যাপ্লায়েন্স বা আপনার পুরো সার্ভারের একটি "সফটওয়্যার ব্লু-প্রিন্ট"। আপনি যখন একটি EC2 সার্ভার তৈরি করেন, তখন এই AMI থেকেই অপারেটিং সিস্টেম এবং অন্যান্য কনফিগারেশন লোড হয়।

## ১. AMI এর মূল কাজ কী?
সহজ কথায়, এটি আপনার সার্ভারের একটি Snapshot বা ক্লোন।

- অপারেটিং সিস্টেম: এতে উবুন্টু, লিনাক্স বা উইন্ডোজের মতো ওএস থাকে।
- অ্যাপ্লিকেশন এবং কনফিগ: আপনি যদি Nginx বা Node.js ইনস্টল করার পর একটি AMI বানান, তবে ভবিষ্যতে ওই AMI থেকে নতুন সার্ভার খুললে সবকিছু আগে থেকেই ইনস্টল করা পাবেন।
- ডিপ্লয়মেন্ট গতি: এটি ব্যবহার করে হাজার হাজার একই ধরণের সার্ভার কয়েক মিনিটের মধ্যে লঞ্চ করা সম্ভব, যা আপনার Auto Scaling-এর জন্য অপরিহার্য।

## ২. AMI এবং EBS এর সম্পর্ক
AMI সরাসরি আপনার EBS (Elastic Block Store) এর স্ন্যাপশটের ওপর ভিত্তি করে তৈরি হয়।

- আপনার বিলে যে EC2-Instances ($0.14) এবং অন্যান্য চার্জ দেখছেন, তার একটি অংশ আপনার তৈরি করা কাস্টম AMI এর জন্য হতে পারে।
- আপনি যখন "Create Image" দেন, তখন এটি আপনার EBS ড্রাইভের একটি কপি বা Snapshot তৈরি করে S3 স্টোরেজে রেখে দেয়।

## ৩. AMI এর প্রকারভেদ

- AWS Marketplace AMIs: এখানে বিভিন্ন কোম্পানি (যেমন- Bitnami, WordPress) আগে থেকে কনফিগার করা ইমেজ দিয়ে রাখে।
- Community AMIs: অন্যদের শেয়ার করা ফ্রি ইমেজ।
- My AMIs (Custom): এটি আপনার জন্য সবচেয়ে গুরুত্বপূর্ণ। আপনি আপনার নিজের পোর্টফোলিও বা ChatApp সেটআপ করার পর যে ইমেজটি বানিয়েছেন।
