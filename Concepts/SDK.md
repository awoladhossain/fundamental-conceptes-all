# SDK (Software Development Kit)

SDK হলো একটি "টুলবক্স" বা সরঞ্জাম সেট যা ডেভেলপারদের নির্দিষ্ট প্ল্যাটফর্ম বা অ্যাপের জন্য সফটওয়্যার তৈরি করতে সাহায্য করে। ধরুন, আপনি একটি ফার্নিচার বানাবেন। SDK হলো সেই প্যাকেজ যেখানে কাঠ, করাত, হাতুড়ি, এবং একটি গাইডবুক (Manual) একসাথে দেওয়া থাকে।

## SDK-র প্রকারভেদ:

- Platform SDKs: যেমন Android SDK বা iOS SDK (মোবাইল অ্যাপ তৈরির জন্য)।
- Cloud SDKs: যেমন AWS SDK বা Google Cloud SDK (ক্লাউড সার্ভিসের সাথে কাজ করার জন্য)।
- Language SDKs: যেমন Java JDK বা .NET SDK।
- Feature SDKs: যেমন Facebook SDK (অ্যাপে ফেসবুক লগইন বা শেয়ারিং যুক্ত করতে)।

## SDK কীভাবে কাজ করে?

SDK-র ভেতরে লাইব্রেরি, ডকুমেন্টেশন, কোড স্যাম্পল এবং ডিবাগিং টুলস থাকে। যখন আপনি কোনো SDK ইন্সটল করেন, আপনি মূলত সেই প্ল্যাটফর্মের তৈরি করা প্রি-বিল্ট ফাংশনগুলো ব্যবহার করার সুযোগ পান, ফলে আপনাকে স্ক্র্যাচ থেকে কোড লিখতে হয় না।


# SSH (Secure Shell) কী?

SSH হলো একটি নেটওয়ার্ক প্রোটোকল যা আপনাকে ইন্টারনেটের মাধ্যমে অন্য একটি কম্পিউটার বা সার্ভারের সাথে নিরাপদে এবং এনক্রিপ্টেড ভাবে যোগাযোগ করতে সাহায্য করে।

## এটি কীভাবে কাজ করে?

SSH মূলত Client-Server Model-এ কাজ করে।

- আপনার কম্পিউটার (Client) থেকে কমান্ড দেন।
- SSH সেই কমান্ডকে এনক্রিপ্ট (লক) করে ইন্টারনেটে পাঠায়।
- রিমোট সার্ভার (Server) সেই লক খুলে কমান্ডটি এক্সিকিউট করে।

এটি সাধারণত Public-Private Key Pair ব্যবহার করে কাজ করে, যা হ্যাকারদের জন্য পাসওয়ার্ড চুরি করা প্রায় অসম্ভব করে তোলে।


## Example

**আপনি একটি 'Weather App' বানাচ্ছেন।**

- SDK: আপনি 'Android SDK' ব্যবহার করে অ্যাপের বাটন, লেআউট এবং ডিজাইন তৈরি করলেন।
- API: এখন আপনার দরকার লাইভ আবহাওয়া। আপনি 'OpenWeatherMap API' ব্যবহার করে সার্ভার থেকে ডেটা আনলেন এবং অ্যাপে দেখালেন।
- SSH: আপনার অ্যাপের ডেটাগুলো একটি লিনাক্স সার্ভারে রাখা আছে। আপনি আপনার বাসা থেকে ল্যাপটপের মাধ্যমে ওই সার্ভারে ঢুকে কোড আপডেট বা ডেটাবেস চেক করার জন্য SSH ব্যবহার করলেন।

### সারমর্ম:

- SDK দিয়ে অ্যাপ তৈরি করেন।

- API দিয়ে অন্য সার্ভিস থেকে ডেটা আনেন।

- SSH দিয়ে দূরবর্তী সার্ভার নিয়ন্ত্রণ করেন।
